# ADR-0013: IP同定にTwitter（X）コーパスを活用する

- **Status**: Accepted (仮説検証中)
- **Date**: 2026-04-25

## 背景

ip-link-agent（Claude Haiku）によるLLMベースのIP同定に精度問題があった。
同系統のアイドルIPを混同するケースが頻発（例: Ave Mujica → ラブライブ！と誤認定）。
LLMはイベントタイトルの表記ゆれや略称に弱く、確信が持てない場合でも何らかのIPを返してしまう傾向がある。

## 決定

イベントタイトルで検索したXのポストをraw収集し、そのBoWをIP同定の補助シグナルとして使う。

## 根拠

**Twitterコーパスはオタクコミュニティに精通している。**

一般的なニュース記事や辞書ベースのコーパスと異なり、Xのオタクユーザーは：

- IP名・キャラ名・略称・界隈用語を自然に使って投稿する
- 公式より早くイベント情報を拡散する
- 同一IPのコンテンツを文脈ごとまとめて言及する（「ハイキュー!!展いく」「烏野ファンとして」）

→ 一般記事コーパスより、IPの実態に即した語彙分布が得られる。

## 仮説: IDF分析でIP名が浮上する

イベント群のツイートBoWにTF-IDFをかけると：

- 「東京」「ライブ」「チケット」などの汎用語 → 多くのイベントに出現 → IDF低 → 沈む
- 大物声優名 → 複数IP跨ぎで出現 → IDF中程度
- **IP名（「ハイキュー!!」「Ave Mujica」等）→ そのIPのイベントにしか出現しない → IDF高 → 浮上する**

IP名は異なるIPのイベント間では基本的に共起しないため、IDFによる弁別が機能するという仮説。

先行研究は確認できていないが、オタク向けイベントドメイン特有の性質（IPの排他性）に基づく。

## トレードオフ

**採用の理由:**
- LLM依存を減らし統計的・再現可能なIP同定ができる
- データが増えるほど精度が上がる自己強化構造
- Twitter APIのコストはあるが、一般記事クロールよりオタク向けドメインへの適合度が高い

**リスク:**
- 仮説が外れる可能性（先行研究なし）
- Twitter API制限によりデータ収集速度に上限がある
- ツイートをイベントタイトルで検索しているため、タイトルに含まれる語が自明に高頻度になるバイアスがある

## 検証方法

`event_keywords`（抽出済み固有名詞）のイベントまたがり分布を集計し、
IP名が汎用語より高IDFで浮上するかどうかを確認する。

## 運用ルール

### キーワード抽出はAPI呼び出しコストがかかる（一回性）

`event_keywords` のキーワード名はClaude Haiku（Gateway経由）が抽出したもの。
**weightの計算式変更など、ローカルで再計算できる変更の場合はテーブルをDELETEして再抽出しない。**

- キーワード名（Claude抽出）→ イベントあたり1回限り、再抽出は最終手段
- weight（TF計算）→ `tweets` テーブルから `UPDATE` で再計算可能

変更内容がキーワード名に影響しない場合は、SQLのUPDATE文で対応する。
