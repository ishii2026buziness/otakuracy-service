# ADR-0005: フリーミアム + API従量課金のマネタイズモデルを採用する

- **Status**: Accepted
- **Date**: 2026-03-21

## Context

オタクイベントキュレーションサービスとして収益化を目指す。
ドメインの特性：
- イベント情報自体は公開情報（希少性はない）
- 価値は「網羅性」「鮮度」「パーソナライズ」にある
- 一元管理されたサイトが存在しないため、収集・整理自体が参入障壁

## Decision

3層のマネタイズモデルを採用：

**Free（集客・認知）**
- イベント一覧の閲覧（日時・場所・内容）
- 基本カテゴリ/エリアフィルタ

**Premium（サブスク）**
- お気に入りIP無制限
- 複合条件の通知アラート（IP × エリア × カテゴリ）
- 販売開始・抽選締切の先行通知
- パーソナルフィード・レコメンド

**API（従量課金）**
- データ提供（APIキー認証）
- AIの進歩でデータ需要が増えることを見込む
- データ品質が上がるほど単価を上げられる構造

課金は「データ量」ではなく「通知精度と探索効率」で成立させる。

## 却下した案

- **広告のみ**: 収益の天井が低く、ユーザー体験を損なう
- **全機能有料**: 集客できない。無料でデータの価値を体験させることが必要

## Consequences

- 無料ユーザーを増やすためにデータの量・鮮度が重要
- Premium の価値を出すために `first_seen_at` / `sales_start_at` / 通知インフラが必須
- API 公開時にスクレイピング防御が必要（自社データの無断利用を防ぐ）
