# ADR-0008: イベント同一性判定を日付+会場ベースに変更する

- **Status**: Accepted
- **Date**: 2026-03-21
- **Supersedes**: dedup_v2 の初期実装（タイトル類似度主体）

## Context

dedup_v2 の初期実装はイベント同一性を「タイトル類似度（Jaccard ≥ 0.8）AND 日付前方一致」で判定していた。

問題：
- タイトルは表記ゆれが激しい。e+ と Eventernote で同一イベントが「TV アニメ【推しの子】体験展」と「推しの子 体験展」のように異なる表記で登録される
- normalize_title が括弧内テキストを削除するため、IP名が括弧内にある場合にタイトルが崩壊する（「TV アニメ【推しの子】体験展」→「tv アニメ体験展」）
- 結果として cross-source merge が機能せず、同一イベントが複数ソース分だけ event テーブルに重複登録される

根本的な問題は**タイトルに依存していること**。タイトルは揺れる。日付と会場は揺れない。

## Decision

イベント同一性の主キーを **日付 + 会場名（正規化）** とする。タイトルは補助的な参照情報に留める。

```
同一イベント判定ルール:
  date（YYYY-MM-DD）が一致
  AND normalize_venue(venue) が一致（or 類似度 ≥ 0.9）
  → 同一イベントとみなす（タイトルは不問）

クロスソースマージの優先度:
  1. date + venue 一致 → 確実に同一（conf=1.0）
  2. date + title類似（≥0.7） → 高確率同一（conf=0.8）
  3. それ以外 → 別イベント
```

会場の正規化（normalize_venue）:
- 全角→半角 NFKC
- 「ホール」「劇場」「アリーナ」等のサフィックスを正規化
- 空白・記号除去

## 却下した案

- **タイトル類似度を主キーにする**: 表記ゆれに弱い。normalize 処理が IP 名を消してしまうリスクがある
- **URL ベース**: ソースが異なれば URL が違うため cross-source merge に使えない

## Consequences

- dedup_v2.py の `dedup_within_source` / `merge_across_sources` を再実装
- 日付・会場の両方が取れないレコードは同一性判定が弱くなる（Eventernote は会場が取れるが e+ は会場 + 都道府県で若干形式が異なる）
- IP 識別（resolve_ip）とは分離された問題。dedup はあくまで「同一イベントの重複除去」であり、どの IP のイベントかは別ステージで判定する
