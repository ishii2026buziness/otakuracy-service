# ADR-0008: イベント同一性判定を日付+会場+IPベースに変更する

- **Status**: Accepted
- **Date**: 2026-03-21
- **Supersedes**: dedup_v2 の初期実装（タイトル類似度主体）

## Context

dedup_v2 の初期実装はイベント同一性を「タイトル類似度（Jaccard ≥ 0.8）AND 日付前方一致」で判定していた。

問題：
- タイトルは表記ゆれが激しい。e+ と Eventernote で同一イベントが「TV アニメ【推しの子】体験展」と「推しの子 体験展」のように異なる表記で登録される
- normalize_title が括弧内テキストを削除するため、IP名が括弧内にある場合にタイトルが崩壊する（「TV アニメ【推しの子】体験展」→「tv アニメ体験展」）
- 結果として cross-source merge が機能せず、同一イベントが複数ソース分だけ event テーブルに重複登録される

さらに「日付+会場一致 → 同一」だけでは不十分：
- 同日・同会場で複数の異なるIP（作品）のイベントが開催される場合がある
- IP が特定できていない状態では「同一イベント」の判断根拠がない

根本的な問題は**タイトルに依存していること**と**IPなしで判定しようとしていること**。タイトルは揺れる。日付と会場は揺れない。IPが違えば別イベント。

## Decision

イベント同一性の主キーを **日付 + 会場名（正規化） + IP** とする。

```
同一イベント判定ルール:
  前提: 両レコードの ip_id が特定済み（non-NULL）かつ一致
  AND date（YYYY-MM-DD）が一致
  AND normalize_venue(venue) が一致
  → 同一イベントとみなす（タイトルは不問）

フォールバック（会場表記がソース間で異なる場合）:
  ip_id が一致 AND date が一致 AND title_similarity ≥ 0.6
  → 同一イベントとみなす

IP が不明（NULL）な場合:
  → 同一性判定不能。dedup 対象外。
  IP 不明イベントは persist されるが event_ip_link が空となり、サービス表示対象外。
```

会場の正規化（normalize_venue）:
- 全角→半角 NFKC
- 空白・記号除去
- lowercase

## 却下した案

- **タイトル類似度を主キーにする**: 表記ゆれに弱い。normalize 処理が IP 名を消してしまうリスクがある
- **URL ベース**: ソースが異なれば URL が違うため cross-source merge に使えない
- **日付+会場のみ（IP不問）**: 同日・同会場で別 IP のイベントが開催されるケースで誤マージが発生する。IP 不明の場合は判定根拠がないため除外すべき

## Consequences

- dedup_v2.py の `_same_event()` は ip_id（両方 non-NULL かつ一致）を必須条件とする
- IP抽出（ADR-0009）が dedup より前に必要（パイプライン順序: discover → extract_ip → dedup）
- IP不明イベントが一定数発生する（初期は特に多い）。比率をモニタリング指標にする
- 日付・会場の両方が取れないレコードも同一性判定が弱くなる
