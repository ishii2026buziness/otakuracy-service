# ADR-0004: 初期ストレージに SQLite を選択する

- **Status**: Accepted
- **Date**: 2026-03-21

## Context

初期実装は `whitelist.json` と `events_processed.json` のJSONファイルのみ。
増分更新・検索・品質メトリクス・通知トリガーが不可能な構造。

DBの選択肢：
1. SQLite
2. PostgreSQL
3. 当面JSONファイルのまま

## Decision

**SQLite** を選択する。

理由：
- K12単体運用、チームなし、まず動かすフェーズ
- PostgreSQL は運用コスト（プロセス管理・バックアップ設計）が早すぎる
- JSONより検索・増分更新・外部キー制約が使える
- APIを公開する段階で PostgreSQL に移行すればよい

## 却下した案

- **PostgreSQL**: 機能的には理想だが現フェーズでは過剰。ユーザーがいない状態で運用コストを払うべきではない。
- **JSONファイル継続**: 検索不可・増分更新不可・品質メトリクス不可。課金サービスの基盤にならない。

## Consequences

- シングルファイルDBなのでK12へのデプロイがシンプル
- 将来的にPostgreSQLへ移行する際にマイグレーションが必要
- 同時書き込みが増えた場合にボトルネックになりうる（その時点で移行判断）
