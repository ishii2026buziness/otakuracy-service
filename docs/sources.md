# ソースカタログ & 評価設計

> 最終更新: 2026-03-21

---

## 評価指標

ソースを採用・除外・優先度付けするための共通指標。

| 指標 | 説明 | 評価 |
|---|---|---|
| **カバレッジ** | イベント件数・カテゴリ・地域の幅 | 広/中/狭 |
| **構造化度** | 日時・会場・価格がHTMLタグで構造化されているか | 高/中/低 |
| **安定性** | HTML構造の変更頻度、スクレイピングの壊れやすさ | 高/中/低 |
| **収集コスト** | requests で取れるか Playwright 必須か | 低/中/高 |
| **鮮度** | 新着イベントが何日後に掲載されるか | 即日/数日/不明 |
| **ToS** | スクレイピング禁止か、robots.txt の状況 | OK/グレー/NG |
| **価格取得可否** | 価格情報が取れるか | ○/△/× |

---

## 収集手段の定義

| 手段 | 説明 | コスト |
|---|---|---|
| **requests** | 通常のHTTPスクレイピング | 低 |
| **Playwright** | JSレンダリング必須サイト向け | 中 |
| **API** | 公式API・GraphQL | 低〜中 |
| **Agent** | Playwright + claude -p での非構造探索 | 高 |

## 現状ソース一覧

### e+ (eplus.jp)
- **URL**: https://eplus.jp/sf/event/anime/
- **カバレッジ**: アニメ系イベント・東京中心（現在東京のみ実装）
- **構造化度**: 高（タイトル・日付・会場・都道府県・販売状態が構造化）
- **安定性**: 中（定期的にHTML変更あり）
- **収集コスト**: requests（低）
- **鮮度**: 即日〜数日
- **ToS**: グレー
- **価格取得**: △（販売状態は取れるが価格テキストは要パース）
- **現状の使われ方**: ホワイトリスト用途のみ ← **プライマリに昇格すべき**
- **課題**: 東京アニメ系のみ。全国・ゲーム・VTuberカテゴリは未対応

### Eventernote
- **URL**: https://www.eventernote.com
- **カバレッジ**: 声優・アーティスト系イベント（ライブ・舞台挨拶中心）
- **構造化度**: 中（タイトル・日付・会場は取れる。価格は薄い）
- **安定性**: 中
- **収集コスト**: requests（低）
- **鮮度**: 即日
- **ToS**: グレー
- **価格取得**: ×（ほぼなし）
- **現状の使われ方**: 声優名の抽出のみ ← **イベントデータとして使うべき**
- **課題**: 声優ベースのため、IPタグ付けにAIが必要

### AniList
- **URL**: https://graphql.anilist.co
- **カバレッジ**: アニメ・マンガのIP情報（イベント情報はなし）
- **構造化度**: 高（GraphQL API）
- **安定性**: 高
- **収集コスト**: API（低）
- **価格取得**: × （イベントデータではない）
- **現状の使われ方**: IP名・公式URLの発見
- **役割**: IP discovery のみ（Tier 3）

### アニメイト商品ランキング
- **URL**: https://www.animate.co.jp/catalog/rank/
- **カバレッジ**: 現在人気IPの商品（イベント情報はなし）
- **構造化度**: 中
- **収集コスト**: requests（低）
- **価格取得**: ×

- **役割**: IP discovery のみ（Tier 3）。アクティブ判定には使えない

### VTuber（ハードコード）
- **現状**: ホロライブ・にじさんじの2グループのみ固定
- **課題**: VSPO・774inc・ぶいすぽっ！等未対応

### 公式サイト（Playwright + claude -p）
- **カバレッジ**: ホワイトリスト登録済みIPの公式サイト全般
- **構造化度**: 低（サイトによって完全に異なる）
- **収集コスト**: Agent（高）
- **価格取得**: △（記載があればAgentが拾う）
- **現状の使われ方**: メインの収集ソース ← **サブ確認用に格下げすべき**
- **役割**: active IPのみ週次確認。チケットサイト未掲載イベントの補完

## ソース採用基準

以下を満たすものを採用対象とする。

**必須:**
- イベントタイトル + 日付 が取れる
- 定常的に更新されている（月1回以上）
- ToSが明示的にNGでない

**加点:**
- 会場・地域が取れる
- 価格・販売状態が取れる
- 構造化度が高い（AI不要）
- APIが公開されている

**除外:**
- ToSで明示的にスクレイピング禁止
- 収集コストに対してカバレッジが低すぎる
- HTMLが頻繁に変わり保守コストが高い

## プラガブル設計方針

ソースを差し替えやすくするためのインターフェース設計。

```python
class EventSource(ABC):
    """全収集ソースの基底クラス"""

    SOURCE_ID: str          # 一意のID ("eplus", "eventernote" 等)
    TIER: int               # 1=プライマリ, 2=補助, 3=IP発見
    COLLECTION_METHOD: str  # "requests" | "playwright" | "api" | "agent"

    @abstractmethod
    def collect_raw(self) -> list[RawEventRecord]:
        """生データを取得して返す。構造化・AI処理はしない。"""
        ...

    def health_check(self) -> SourceHealth:
        """疎通確認。CI/監視で定期実行。"""
        ...
```

```python
@dataclass
class RawEventRecord:
    """スクレイピング直後の生データ。必ずsource_urlとraw_*を保持。"""
    source_id: str
    source_url: str
    fetched_at: datetime
    raw_title: str
    raw_date_text: str | None
    raw_venue_text: str | None
    raw_price_text: str | None
    raw_body: str | None        # AI抽出用の生テキスト
    structured_fields: dict     # 構造化済みフィールド（あれば）
```

ソースの追加は `EventSource` を継承したクラスを作るだけ。
ソースのon/offは設定ファイルで制御。パイプライン本体はソースを意識しない。
